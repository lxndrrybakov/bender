# Bender

Система Bender — это интерфейс и API для автоматизации поздравлений клиентов риелторского агентства. Текущая версия работает на синтетических данных и демонстрирует ключевые сценарии: просмотр отправленных и запланированных поздравлений, создание письма вручную, генерация текста с помощью нейросети и отправка через Resend.

## Структура

- `client/` — фронтенд на React + Vite. Используется React Query для работы с API и Tailwind CSS для стилизации.
- `server/` — Node.js API без базы данных. Содержит синтетическое хранилище, генерацию текста (Hugging Face Inference) и интеграцию с Resend.

## Запуск

> ⚠️ В текущем окружении зависимости не установлены. Перед запуском выполните установку в обеих папках (`client` и `server`) или с помощью npm workspaces из корня.

```bash
# установка зависимостей
npm install
npm install --workspace server
npm install --workspace client

# запуск разработки (клиент + сервер)
npm run dev

# предпросмотр только фронтенда
npm run preview:client
```

Сервер поднимется на `http://localhost:4000`, клиент — на `http://localhost:5173`.

### Переменные окружения сервера

Создайте файл `server/.env` на основе `server/.env.example`:

```
RESEND_API_KEY=ваш_ключ_resend
RESEND_FROM_EMAIL=адрес_отправителя
HF_API_TOKEN=токен_hugging_face
PORT=4000
```

- При отсутствии `RESEND_API_KEY` отправка письма будет имитироваться (результат сохраняется, но письмо не уходит).
- При отсутствии `HF_API_TOKEN` текст поздравления формируется шаблоном.

## Основные возможности

- 📬 **Журнал отправленных писем** — список с деталями письма и статусом доставки.
- ⏳ **Предстоящие поздравления** — события с расчётом дней до отправки.
- ✍️ **Ручная отправка** — форма для моментальной отправки или планирования письма.
- 🤖 **Генерация текста** — интеграция с Hugging Face (Falcon 7B Instruct) и fallback шаблон.
- 📡 **Интеграция с Resend** — отправка HTML-писем или вывод черновика в консоль при отсутствии ключа.
